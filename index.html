How to Add Your Data for Hosting on the Website

  • Visit the repository at https://github.com/kerzendorf-group/group-data and fork it.
  • There exists a template_person direcotry inside members directory. Simple follow the same format to update your own directory. Kindly change your pre existing structure so that it looks like the one of template_person

    • Detailed steps

  1. Create a new directory with your name (if it doesn't exists already) and inside your directory's json folder copy all json files from template_person/jsons and enter your own details in those json files.
  2. The values of keys in social_links.jsons can be "None" (N uppercase) in case you don't have anything to enter.Example "orchid": "None"
  3. In case you don't have anything to enter inside any other json file, example you don't have "books" or "posters" then erase that section from json while making sure syntax is correct.
  4. Please note that the “category” key inside basic_data.json should be spelled same “Graduates”, “Undergraduates”, “Post Graduates”, “Post Doctorates” only, in order to group people together. Please add "myrole": "Your_role" inside your JSON file as the value for your latest role in group.
  5. It is important to note that if you are currently a part of kerzendorf group then in basic_info.json the value "group_end_year": "Present" should exist, else enter year of leaving so that your name can be placed in Alumni category
  6. Upload your profile picture (square-shaped), which will be displayed on the website, into the media/images directory.
  7. Upload any other necessary media into the media folder and any documents into the documents directory.
  8. Ensure that after uploading the files, you update those changes in your own JSON files present in your directory as well, with the path pointing to the file location. The path to files present inside the media or document directory should start with members/data followed by your directory name and then the rest of the path.
  9. In the groupwebsite_data directory, open people_list.json and make an entry inside "people" and "people_directory" keys. By looking at already present entries in the list
  10. Finally, create a Pull Request.
